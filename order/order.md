# 订单
常用操作
--------------------------------------------------------------------------

### 作废

作废操作(`order/discard`), 既能让订单消失又能保留订单记录的一种操作。使用 Edition 承载.

满足以下任意一种情况都无法使用作废操作：

- 订单已开始交付（存在交付记录）；
- 存在取料记录；
- 存在采购行为(不能含有该订单的采购清单)；
- 样品订单关联的使用反馈没有完成反馈上报；

作废操作评审完成后，将执行以下操作：

1. 将订单状态设置为“已作废”；
2. 将 `order.payment_status` 设置为“未付款”；
3. 删除付款记录；
4. 删除先发货后付款记录 (如果有的话);
